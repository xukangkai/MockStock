技术实现方式决定了仓鼠长什么样、动起来怎么样

大 emoji 组合 + CSS 缩放动画

A. Emoji 组合

开发最快，几小时就能跑起来。缺点：emoji 风格固定，胖瘦靠缩放，表现力有限

优点

  • 开发极快
  • 无需美术资源
  • 跨平台兼容

缺点

  • 外观不可定制
  • 动画僵硬
  • 进化形态靠 emoji 拼接，略显粗糙
→
CSS 绘制像素仓鼠，完全自定义

B. CSS/SVG 绘制

用代码画仓鼠，每个状态都是独立的图形。胖瘦、表情、装备全可控

优点

  • 外观完全可控
  • 动画流畅
  • 无需外部图片

缺点

  • 开发时间较长
  • CSS 画图表现力有限
  • 复杂形态难实现

Canvas/字符画，可动画可交互

C. Canvas 动画

HTML5 Canvas 实时绘制，支持帧动画、物理效果（弹跳、摇晃）

优点

  • 动画最流畅
  • 可做复杂动作
  • 交互性强

缺点

  • 开发最复杂
  • 需要像素图素材
  • 代码量大
